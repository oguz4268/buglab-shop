Wichtige UI-Flows

Flow Start Schritte Erwartung
Produkt suchen /products Suchbegriff eingeben, Anwenden klicken Produktliste zeigt passende Treffer und Trefferanzahl
Kategorie filtern /products Kategorie auswählen oder Schnellfilter klicken Nur Produkte der ausgewählten Kategorie werden angezeigt
Produktdetail öffnen /products Details ansehen klicken Produktdetailseite zeigt Name, Preis, Rating, Lagerbestand, Lieferung
Warenkorb /products Verfügbares Produkt in Warenkorb legen Produkt erscheint im Warenkorb mit Menge, Einzelpreis und Zeilensumme
Nicht verfügbares Produkt /products Laptop Stand prüfen Produkt ist sichtbar, aber nicht bestellbar
Checkout /cart Zur Kasse, Formular gültig ausfüllen, Bestellung abschließen Bestellbestätigung zeigt Bestellnummer, Name, E-Mail, Artikelanzahl, Gesamtbetrag
Login /login Demo-Zugang eingeben Konto-Seite zeigt Name, E-Mail und Rolle

API-Endpunkte

Methode Endpoint Beschreibung Erwartung
GET /api/products Alle Produkte HTTP 200, JSON-Liste mit allen Produkten
GET /api/products?q=usb Produktsuche HTTP 200, passende Produkte
GET /api/products?category=Audio Kategorie-Filter HTTP 200, nur Audio-Produkte
GET /api/products?q=usb&category=Adapter%20%26%20Hubs Kombinierte Suche und Kategorie HTTP 200, USB-C Hub
GET /api/products/1 Produkt per ID HTTP 200, einzelnes Produkt
GET /api/products/999 Nicht vorhandenes Produkt HTTP 404, Fehlercode PRODUCT_NOT_FOUND
POST /api/checkout/validate Checkout-Daten validieren HTTP 200 bei gültigen Daten, HTTP 400 bei ungültigen Daten

cURL-Beispiele

curl "http://localhost:8080/api/products"

curl "http://localhost:8080/api/products?category=Audio"

curl "http://localhost:8080/api/products?q=usb&category=Adapter%20%26%20Hubs"

curl -i "http://localhost:8080/api/products/999"

Checkout-API gültig

curl -X POST "http://localhost:8080/api/checkout/validate" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "Max",
    "lastName": "Mustermann",
    "email": "max@example.com",
    "street": "Musterstraße 1",
    "postalCode": "12345",
    "city": "Berlin",
    "paymentMethod": "invoice"
  }'

Checkout-API ungültig

curl -i -X POST "http://localhost:8080/api/checkout/validate" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "M",
    "lastName": "",
    "email": "invalid",
    "street": "",
    "postalCode": "12",
    "city": "",
    "paymentMethod": ""
  }'
